# 数组遍历与 for-each

遍历就是「逐个访问数组的每个元素」。Java 有两种方式：索引 for 循环和增强 for（for-each）循环。

## 索引 for 循环

用索引变量从 0 走到 `length-1`：

```java
int[] a = {3, 1, 4, 1, 5};
for (int i = 0; i < a.length; i++) {
    System.out.println(i + ": " + a[i]);
}
```

特点：

- 能拿到**索引 `i`**，可访问相邻元素、做条件定位。
- **可以修改**数组元素：`a[i] = a[i] * 2;`。

## 增强 for 循环（for-each）

直接取出每个元素，**无需索引**：

```java
int[] a = {3, 1, 4, 1, 5};
for (int x : a) {              // 读作「对 a 中的每个 int x」
    System.out.println(x);
}
```

特点：

- 语法简洁，适合「只读地遍历每个元素」。
- **拿不到索引**。
- **不能修改数组元素本身**——`x` 是元素的副本。

## 关键区别：for-each 改不动数组

```java
int[] a = {1, 2, 3};
for (int x : a) {
    x = x * 2;          // 只改了副本 x，数组 a 不变！
}
// a 仍是 {1, 2, 3}

for (int i = 0; i < a.length; i++) {
    a[i] = a[i] * 2;    // ✔ 通过索引才能真正修改
}
// a 变成 {2, 4, 6}
```

> 对**对象数组**，for-each 变量是引用的副本，因此可以**调用方法改对象状态**，但不能让数组元素指向新对象。

## 如何选择

| 需求 | 用哪种 |
| --- | --- |
| 只是逐个读元素 | for-each（更简洁） |
| 需要索引（位置、相邻元素） | 索引 for |
| 要修改数组元素的值 | 索引 for |
| 反向 / 跳着遍历 | 索引 for |

## 遍历部分元素

只需索引 for 调整边界：

```java
// 遍历前一半
for (int i = 0; i < a.length / 2; i++) { ... }
// 反向遍历
for (int i = a.length - 1; i >= 0; i--) { ... }
```

## 易错提醒

- 想修改数组却用了 for-each → 数组没变（最常见错误）。
- for-each 中无法访问索引，需要位置信息时别用它。
- 索引 for 边界用 `< a.length`，避免越界。

## 小结

- 两种遍历：**索引 for**（有索引、可改、灵活）与 **for-each**（简洁、只读基本类型元素）。
- for-each 改不动基本类型数组元素；要修改用索引 for。
- 按「是否需要索引 / 是否要修改」来选。
