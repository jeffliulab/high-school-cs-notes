# 逻辑运算符与德摩根定律

逻辑运算符 `&&`、`||`、`!` 把多个布尔条件组合起来；短路求值与德摩根定律是高频考点。

## 三个逻辑运算符

| 运算符 | 名称 | 真值规则 |
| --- | --- | --- |
| `&&` | 与 (AND) | 两边都 `true` 才 `true` |
| `\|\|` | 或 (OR) | 任一为 `true` 即 `true` |
| `!` | 非 (NOT) | 取反 |

```java
boolean ok = (age >= 13) && (age <= 19);   // 13~19 之间
boolean weekend = day.equals("六") || day.equals("日");
boolean notDone = !done;
```

## 真值表

| A | B | `A && B` | `A \|\| B` | `!A` |
| --- | --- | --- | --- | --- |
| T | T | T | T | F |
| T | F | F | T | F |
| F | T | F | T | T |
| F | F | F | F | T |

## 短路求值（Short-circuit）

Java 从左到右求值，**一旦能确定结果就停止**：

- `A && B`：若 `A` 为 `false`，**不再求** `B`（结果必为 false）。
- `A || B`：若 `A` 为 `true`，**不再求** `B`（结果必为 true）。

这不仅是效率优化，更能**防止崩溃**：

```java
// 若 s 为 null，先判断 s != null 可避免 NullPointerException
if (s != null && s.length() > 0) { ... }   // 安全

// 防止除以零
if (b != 0 && a / b > 2) { ... }
```

把「保护性条件」放在 `&&` 左边是常用技巧。

## 运算优先级

`!` > 关系运算符（`< > == ...`）> `&&` > `||`。

```java
boolean r = a > 0 && b > 0 || c > 0;
// 等价于 ((a>0) && (b>0)) || (c>0)
```

不确定时**多加括号**。

## 德摩根定律（De Morgan's Laws）

把「非」分配到括号内时，`&&` 与 `||` 互换：

$$\lnot(A \land B) = \lnot A \lor \lnot B$$
$$\lnot(A \lor B) = \lnot A \land \lnot B$$

```java
!(x > 0 && y > 0)   等价于   (x <= 0 || y <= 0)
!(a == b || c == d) 等价于   (a != b && c != d)
```

> 取反时记住两步：**`&&`↔`||` 互换，每个子条件各自取反**。这是 MCQ「下列哪个表达式与……等价」的标准考法。

## 易错提醒

- `!(a && b)` ≠ `!a && !b`；正确是 `!a || !b`。
- 取反关系运算符：`>` 的反是 `<=`（不是 `<`）。
- 依赖短路顺序时，保护条件必须在左边。

## 小结

- `&& / || / !` 组合布尔条件；记牢真值表。
- **短路求值**既提速又能避免 null/除零崩溃。
- **德摩根定律**：取反时 `&&`↔`||` 且各子条件取反。
