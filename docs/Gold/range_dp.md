# 区间 DP

> **难度**：⭐⭐⭐⭐☆
> **前置知识**：[DP 入门](intro_dp.md)

区间 DP 的状态是一个区间 `dp[l][r]`，通过枚举区间内的**分割点**合并子区间。适合「合并石子」「括号匹配」等问题。

## 核心思想

- 状态：`dp[l][r]` = 处理区间 $[l, r]$ 的最优解。
- 转移：枚举分割点 $k$，把 $[l,r]$ 拆成 $[l,k]$ 和 $[k+1,r]$（或 $[l,k]$ 和 $[k,r]$）：

$$dp[l][r] = \min_{l \le k < r}\big(dp[l][k] + dp[k+1][r] + \text{合并代价}\big)$$

- 计算顺序：**按区间长度从小到大**，保证小区间先算好。

## 模板：按长度枚举

```cpp
// 初始化长度为 1 的区间
for (int i = 0; i < n; i++) dp[i][i] = 0;

for (int len = 2; len <= n; len++)            // 区间长度
    for (int l = 0; l + len - 1 < n; l++) {   // 左端点
        int r = l + len - 1;
        dp[l][r] = INF;
        for (int k = l; k < r; k++)           // 分割点
            dp[l][r] = min(dp[l][r],
                           dp[l][k] + dp[k+1][r] + cost(l, r));
    }
cout << dp[0][n-1];
```

## 经典例：合并石子

> $n$ 堆石子排成一排，每次合并相邻两堆，代价为两堆之和，求合并成一堆的最小总代价。

合并代价用前缀和 $O(1)$ 求区间和：

```cpp
int pre[N];   // 前缀和
auto sum = [&](int l, int r){ return pre[r+1] - pre[l]; };

for (int len = 2; len <= n; len++)
    for (int l = 0; l + len - 1 < n; l++) {
        int r = l + len - 1;
        dp[l][r] = INF;
        for (int k = l; k < r; k++)
            dp[l][r] = min(dp[l][r],
                           dp[l][k] + dp[k+1][r] + sum(l, r));
    }
```

> 合并 $[l,r]$ 的最后一步必然把某个 $[l,k]$ 和 $[k+1,r]$ 合并，合并代价是整段之和 `sum(l,r)`。

## 其他区间 DP 模型

- **括号/回文**：`dp[l][r]` 表示区间能否匹配 / 最长回文子序列。
- **最优三角剖分**：多边形分割，分割点是某个顶点。
- **环形区间**（如环形合并石子）：把数组**复制一份接在后面**变成 $2n$ 长，再做区间 DP。

## 复杂度

状态 $O(n^2)$ × 枚举分割点 $O(n)$ = $O(n^3)$。故 $n$ 一般 $\le 500$。

## 易错提醒

- 必须**按区间长度递增**计算，否则依赖的子区间未就绪。
- 分割点范围 `l <= k < r`，对应 `[l,k]` 与 `[k+1,r]`。
- 区间和用前缀和 $O(1)$，别在内层重新累加。
- 环形问题记得「破环为链」复制数组。

## 小结

- 区间 DP：`dp[l][r]` 枚举分割点合并，$O(n^3)$，$n\le500$。
- 按**区间长度从小到大**计算；合并代价常用前缀和。
- 经典：合并石子、回文、多边形剖分、环形（复制成 2n）。
