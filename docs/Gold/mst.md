# 最小生成树

> **难度**：⭐⭐⭐☆☆
> **前置知识**：[并查集](dsu.md)、[排序入门](../Bronze/intro_sorting.md)

最小生成树（MST）是连通无向图中，连接所有顶点、**总权值最小**的无环边集（$n-1$ 条边）。Kruskal 算法配合并查集是标准解法。

## 问题与性质

给带权连通无向图，选 $n-1$ 条边使所有点连通且总权最小。MST 不含环，是一棵树。常见应用：用最少成本连通所有城市/电缆。

## Kruskal 算法

**贪心**：把所有边按权**从小到大**排序，依次尝试加入——若两端不在同一连通块（不成环）则加入，用**并查集**判断。

```cpp
struct Edge { int u, v, w; };
Edge e[M];

long long kruskal(int n, int m) {
    sort(e, e + m, [](const Edge&a, const Edge&b){ return a.w < b.w; });
    init(n);                         // 并查集初始化
    long long total = 0;
    int cnt = 0;
    for (int i = 0; i < m; i++) {
        if (find(e[i].u) != find(e[i].v)) {   // 不成环
            unite(e[i].u, e[i].v);
            total += e[i].w;
            cnt++;
            if (cnt == n - 1) break;          // 已连成树
        }
    }
    return total;     // 若 cnt < n-1，图不连通
}
```

复杂度 $O(m \log m)$（排序主导）。

## 为什么贪心正确（割性质）

**割性质**：对任意把顶点分成两部分的「割」，跨越割的**最小权边**一定属于某棵 MST。Kruskal 每次加的最小可行边都满足此性质，故得到最优解。

## Prim 算法（点扩展，简述）

另一种 MST 算法：从一个点开始，每次加入「连接已选集合与外部的最小边」，用优先队列实现，$O((V+E)\log V)$。**稠密图**（边多）时 Prim 可能更优；稀疏图 Kruskal 更常用。

## 常见变体

- **判断图是否连通**：MST 加入的边数若 `< n-1` 则不连通。
- **最大生成树**：边按权**降序**排序。
- **次小生成树**、**瓶颈生成树**（最小化最大边）等进阶变体。
- **Kruskal 重构树**（Platinum）：把 Kruskal 过程建成一棵树，处理「路径最大边」类查询。

## 易错提醒

- Kruskal 必须配并查集判环，且边要**升序**排序。
- 加够 $n-1$ 条边即可停止。
- 总权值用 `long long`。
- 图不连通时不存在生成树（cnt 达不到 n-1）。

## 小结

- MST：连通所有点、总权最小的 $n-1$ 条边。
- **Kruskal**：边升序 + 并查集判环，$O(m\log m)$，由**割性质**保证最优。
- 变体：最大生成树（降序）、连通性判断、次小生成树等。
