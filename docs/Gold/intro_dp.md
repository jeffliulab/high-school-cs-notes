# DP 入门

> **难度**：⭐⭐⭐☆☆
> **前置知识**：递归、数组

动态规划（DP）把大问题拆成**重叠的子问题**，每个子问题**只算一次**并保存结果。几乎每场 Gold 都有 DP 题——它是 Gold 的核心。

## DP 的本质

当一个问题满足：

- **最优子结构**：大问题的最优解由子问题的最优解组合而成。
- **重叠子问题**：同一个子问题被反复需要。

就可以用 DP：把子问题答案存进表（数组），避免重复计算。

## 四步思考法

1. **状态定义**：`dp[i]` 表示什么？（最关键）
2. **转移方程**：`dp[i]` 如何由更小的状态算出？
3. **边界/初始化**：最小子问题的答案。
4. **计算顺序**：保证算 `dp[i]` 时它依赖的状态已算好。

## 例：爬楼梯（斐波那契型）

> 每次走 1 或 2 级，到第 $n$ 级有几种走法？

- 状态：`dp[i]` = 到第 $i$ 级的方案数。
- 转移：`dp[i] = dp[i-1] + dp[i-2]`（最后一步走 1 或 2 级）。
- 边界：`dp[0]=1, dp[1]=1`。

```cpp
dp[0] = 1; dp[1] = 1;
for (int i = 2; i <= n; i++)
    dp[i] = dp[i-1] + dp[i-2];
cout << dp[n];
```

## 记忆化搜索（自顶向下）

用递归 + 缓存，等价于 DP，思路更直接：

```cpp
long long memo[N];
bool done[N];
long long solve(int i) {
    if (i <= 1) return 1;
    if (done[i]) return memo[i];      // 已算过，直接返回
    done[i] = true;
    return memo[i] = solve(i-1) + solve(i-2);
}
```

> **递推（自底向上）** 与 **记忆化（自顶向下）** 是 DP 的两种实现，结果相同。状态间依赖复杂时记忆化更省心。

## 例：最大子段和（Kadane）

> 连续子数组的最大和。

- 状态：`dp[i]` = 以 $i$ 结尾的最大子段和。
- 转移：`dp[i] = max(a[i], dp[i-1] + a[i])`（接上前面 or 自己重开）。

```cpp
long long best = a[0], cur = a[0];
for (int i = 1; i < n; i++) {
    cur = max((long long)a[i], cur + a[i]);
    best = max(best, cur);
}
```

## 识别 DP 题的信号

- 求「方案数 / 最大值 / 最小值 / 能否达到」。
- 每一步有若干选择，且选择影响后续。
- 暴力是指数级，但状态数是多项式级。

## 易错提醒

- **状态定义不清**是 DP 出错的头号原因——先把 `dp[i]` 的含义写清楚。
- 初始化和边界要和状态定义一致。
- 计算顺序错（依赖的状态还没算）会得错值。
- 数值大时 `long long`，计数题及时取模。

## 小结

- DP = 最优子结构 + 重叠子问题，每个子问题只算一次。
- 四步：**状态定义 → 转移 → 边界 → 顺序**。
- 递推与记忆化搜索是两种等价实现；先把状态含义想清楚。
