# 网格路径

> **难度**：⭐⭐☆☆☆
> **前置知识**：[DP 入门](intro_dp.md)

二维网格上的 DP：从一角走到另一角，求路径数、最大/最小代价。状态就是格子坐标，转移来自相邻格子，是二维 DP 的入门模型。

## 基本模型：路径计数

> 从左上 $(0,0)$ 走到右下 $(R-1,C-1)$，只能向右或向下，有几条路径？（含障碍）

- 状态：`dp[r][c]` = 到 $(r,c)$ 的路径数。
- 转移：`dp[r][c] = dp[r-1][c] + dp[r][c-1]`（从上或从左来）。
- 边界：`dp[0][0] = 1`；障碍格 `dp[r][c] = 0`。

```cpp
dp[0][0] = (grid[0][0] == '.') ? 1 : 0;
for (int r = 0; r < R; r++)
    for (int c = 0; c < C; c++) {
        if (grid[r][c] == '#') { dp[r][c] = 0; continue; }   // 障碍
        if (r > 0) dp[r][c] += dp[r-1][c];
        if (c > 0) dp[r][c] += dp[r][c-1];
    }
cout << dp[R-1][C-1];
```

## 最小路径和

> 每格有代价，求左上到右下的最小代价路径（只能右/下）。

```cpp
dp[0][0] = cost[0][0];
for (int r = 0; r < R; r++)
    for (int c = 0; c < C; c++) {
        if (r == 0 && c == 0) continue;
        long long best = LLONG_MAX;
        if (r > 0) best = min(best, dp[r-1][c]);
        if (c > 0) best = min(best, dp[r][c-1]);
        dp[r][c] = best + cost[r][c];
    }
cout << dp[R-1][C-1];
```

## 计算顺序很关键

按**行优先、从上到下、从左到右**遍历，能保证算 `dp[r][c]` 时 `dp[r-1][c]` 和 `dp[r][c-1]` 都已算好。

## 变体

- **能向右/下/右下三方向**：转移多加一项 `dp[r-1][c-1]`。
- **收集最多的物品**：`dp[r][c] = max(上, 左) + grid[r][c]`。
- **两条路径同时走**（如「方格取数」）：状态升维成 `dp[r1][c1][r2][c2]`，常压成 `dp[step][c1][c2]`。

## 路径计数取模

路径数可能极大，按题意对 `MOD` 取模：

```cpp
dp[r][c] = (dp[r-1][c] + dp[r][c-1]) % MOD;
```

> 无障碍的纯网格路径数有闭式 $\binom{R+C-2}{R-1}$（见 [组合数学](combinatorics.md)）；有障碍才必须 DP。

## 易错提醒

- 边界格（第一行/第一列）只有一个来源，注意 `r>0`/`c>0` 判断。
- 障碍格的 `dp` 置 0，且不能从它转移。
- 计算顺序必须保证依赖已就绪。
- 计数取模、求和用 `long long`。

## 小结

- 网格 DP：状态 = 坐标，转移来自上方/左方（或加左上）。
- 路径计数 `dp[r][c]=上+左`；最小代价 `min(上,左)+cost`。
- 行优先遍历保证依赖就绪；多路径问题升维。
