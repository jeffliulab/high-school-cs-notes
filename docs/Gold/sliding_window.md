# 滑动窗口

> **难度**：⭐⭐⭐☆☆
> **前置知识**：[双指针](../Silver/two_pointers.md)、[优先队列](../Silver/priority_queues.md)

滑动窗口在长度固定或可变的「窗口」滑过数组时，**高效维护窗口内的聚合量**（和、最大值、不同元素数等）。Gold 常考定长窗口最值——单调队列。

## 定长窗口的和

窗口右移时加新元素、减旧元素，$O(n)$：

```cpp
long long sum = 0;
for (int i = 0; i < n; i++) {
    sum += a[i];
    if (i >= k) sum -= a[i - k];     // 移出窗口左端
    if (i >= k - 1) /* 窗口 [i-k+1, i] 的和是 sum */ ;
}
```

## 定长窗口最大值：单调队列

求每个长度为 $k$ 的窗口的最大值。用**双端队列**维护下标，保持队列中对应值**单调递减**：

```cpp
#include <deque>
deque<int> dq;                       // 存下标，a[dq] 递减
vector<int> res;
for (int i = 0; i < n; i++) {
    // 1) 移除超出窗口左界的下标
    while (!dq.empty() && dq.front() <= i - k) dq.pop_front();
    // 2) 从队尾移除比 a[i] 小的（它们不可能再成为最大值）
    while (!dq.empty() && a[dq.back()] <= a[i]) dq.pop_back();
    dq.push_back(i);
    // 3) 队首即当前窗口最大值
    if (i >= k - 1) res.push_back(a[dq.front()]);
}
```

队首始终是窗口最大值；每个下标进出队各一次，$O(n)$。

> 求窗口**最小值**则维护单调**递增**队列（队尾移除比 `a[i]` 大的）。

## 单调队列 vs 优先队列

| | 单调队列 | 优先队列(堆) |
| --- | --- | --- |
| 定长窗口最值 | $O(n)$ | $O(n\log n)$ |
| 实现 | 双端队列 | `priority_queue` |

定长窗口最值首选单调队列（更快）；不定长或更复杂的可用堆 + 惰性删除。

## 应用：单调队列优化 DP

形如 $dp[i] = \min_{i-k \le j < i}(dp[j]) + c_i$ 的转移，用单调队列维护滑动窗口最小值，把 $O(nk)$ 优化到 $O(n)$。这是 Gold/Platinum DP 优化的常见手段。

## 可变窗口（回顾）

「最长/最短满足条件的子数组」用双指针伸缩窗口（见 [双指针](../Silver/two_pointers.md)），要求性质单调。

## 易错提醒

- 单调队列存**下标**，便于判断是否超出窗口左界。
- 弹出条件用 `<=` 还是 `<` 影响相等元素处理，一般 `<=` 即可。
- 先移除过期队首，再处理队尾，再取答案，顺序别乱。
- 定长窗口的和/最值在 `i >= k-1` 后才开始记录。

## 小结

- 滑动窗口 $O(n)$ 维护窗口聚合量。
- 定长窗口最值用**单调队列**（双端队列，存下标，保持单调）。
- 可优化形如「窗口内取 min/max」的 DP 转移；优于堆的 $O(n\log n)$。
