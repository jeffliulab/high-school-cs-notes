# 分类讨论

> **难度**：⭐⭐☆☆☆
> **前置知识**：基本条件语句

有些题目没有统一公式，需要把问题拆成**若干互不重叠的情形**，每种情形单独处理。关键是做到**不重不漏**。

## 什么是分类讨论

当答案依赖于输入落在哪个「类别」时，就要分类。例如：

- 判断三条边能组成什么三角形（等边/等腰/直角/普通/不能）。
- 两个区间的关系（相离/相交/包含）。
- 棋子能否一步到达（按相对位置分几种走法）。

## 核心原则：不重不漏（MECE）

划分的情形必须：

- **互斥**：任意输入只属于一种情形，不重复计算。
- **完备**：所有可能输入都被覆盖，无遗漏。

漏掉一种情形或两种情形重叠，都会导致错误。

## 例：三角形分类

```cpp
// 已知三边 a <= b <= c（先排序）
int x[3] = {a, b, c};
sort(x, x + 3);
a = x[0]; b = x[1]; c = x[2];

if (a + b <= c) {
    cout << "不能构成三角形\n";
} else if (a == c) {
    cout << "等边\n";
} else if (a == b || b == c) {
    cout << "等腰\n";
} else if (a*a + b*b == c*c) {
    cout << "直角\n";
} else {
    cout << "普通\n";
}
```

> 先排序统一为 `a <= b <= c`，能**大幅减少**要讨论的情形（否则要考虑各种大小顺序）。

## 减少情形数的技巧

- **排序/归一化**：把对称的输入统一成一种顺序（如上例）。
- **取绝对值**：处理对称的正负情形。
- **交换变量**：保证 `a <= b`，省去镜像讨论。

```cpp
if (a > b) swap(a, b);     // 之后只需讨论 a <= b 的情形
```

## 例：两点间国际象棋走法

> 问马/象/车从 A 到 B 各需几步——每种棋子按相对位置分类讨论。

这类题没有捷径，**画图列举**所有相对位置，逐一给出步数，注意「同一格」「同行同列」等边界。

## 易错提醒

- **边界情形**最易漏：相等、为零、共线、重合等。
- 用 `else if` 链保证互斥；若用独立 `if` 要确保情形不重叠。
- 先归一化（排序/交换/取绝对值）能显著简化讨论。
- 写完后用几个极端样例自测每个分支。

## 小结

- 分类讨论 = 把问题拆成**不重不漏**的情形分别处理。
- 排序、交换、取绝对值等归一化手段能减少情形数。
- 重点排查**边界与相等**情形，逐分支测试。
