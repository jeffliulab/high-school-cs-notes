# 基础完全搜索

> **难度**：⭐⭐☆☆☆
> **前置知识**：[时间复杂度](time_complexity.md)、嵌套循环

完全搜索（暴力枚举）= **检查所有可能的情况**，从中找答案。当数据范围小时，它简单、可靠、不易写错——是铜级的主力武器。

## 核心思想

把「所有候选答案」逐一枚举，对每个候选检查是否满足条件。只要候选总数在时限内（对照 [时间复杂度](time_complexity.md) 表），暴力就能过。

## 例 1：枚举一个变量

> 找 $[1, n]$ 中满足某性质的数。

```cpp
for (int x = 1; x <= n; x++) {
    if (check(x)) cout << x << "\n";
}
```

## 例 2：枚举两个数（双层）

> 数组中是否存在两数之和等于 $T$？（$n \le 5000$）

```cpp
bool found = false;
for (int i = 0; i < n; i++)
    for (int j = i + 1; j < n; j++)       // j 从 i+1 起，避免重复
        if (a[i] + a[j] == T) found = true;
```

复杂度 $O(n^2)$，$n=5000$ 时 $2.5\times10^7$，可行。

## 例 3：枚举所有子矩形 / 区间

> 求最大子段和（暴力版，$n \le 1000$）。

```cpp
int best = INT_MIN;
for (int i = 0; i < n; i++) {
    int sum = 0;
    for (int j = i; j < n; j++) {     // 枚举所有以 i 起的区间
        sum += a[j];
        best = max(best, sum);
    }
}
```

## 例 4：枚举坐标 / 配对

网格题枚举每个格子、每对点等，都是完全搜索的变体。关键是想清楚「候选空间是什么」，然后用循环把它走遍。

## 如何判断能否暴力

1. 估计候选总数（枚举层数 → 复杂度）。
2. 对照数据范围与 [时间复杂度](time_complexity.md) 表。
3. 若在 $10^8$ 量级内 → 放心暴力；否则需要更优算法或剪枝。

```
枚举一个数：    O(n)
枚举两个数：    O(n^2)
枚举三个数：    O(n^3)   —— n≤500 可行
枚举所有子集：  O(2^n)   —— 见「递归完全搜索」
```

## 易错提醒

- 枚举无序对时让内层从 `i+1` 开始，避免重复或自配对。
- 暴力前**先估复杂度**，别在 $n=10^5$ 时写 $O(n^2)$。
- 注意初值（如最大值用 `INT_MIN`，最小值用 `INT_MAX`）。

## 小结

- 完全搜索 = 枚举所有候选并逐一检查，小数据下首选。
- 枚举层数决定复杂度：$O(n)$/$O(n^2)$/$O(n^3)$。
- 写之前先用数据范围确认暴力不会超时。
