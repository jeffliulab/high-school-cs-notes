# Ad Hoc 杂题

> **难度**：⭐⭐☆☆☆
> **前置知识**：综合

Ad Hoc（拉丁语「为此」）指**没有固定算法套路**的题——靠观察题目特性、发现规律或巧妙构造来解决。铜级常有一两道，考查思维灵活性。

## Ad Hoc 的特点

- 不属于任何标准算法类别（不是图、不是 DP……）。
- 解法往往简短，但**需要一个关键洞察**。
- 难点在「想到」，而非「实现」。

## 解题思路：如何找到那个洞察

1. **手算小样例**：从 $n=1,2,3$ 开始算，观察答案规律。
2. **找不变量 / 单调性**：什么量始终不变？什么随操作单调变化？
3. **逆向思考**：从目标状态倒推。
4. **对称性 / 奇偶性**：很多 Ad Hoc 题答案取决于奇偶。
5. **暴力打表找规律**：写暴力算出前若干项，在 OEIS 或肉眼找通项。

## 例 1：奇偶性洞察

> 一排灯，每次操作翻转某段状态，问能否达到目标——常归结为某种**奇偶不变量**。

```cpp
// 很多此类题：统计某特征的奇偶，若与目标奇偶不符则不可能
int parity = 0;
for (int i = 0; i < n; i++) parity ^= a[i];
```

## 例 2：找规律 / 数学公式

> 求某种排列/计数的第 $k$ 项——手算前几项，发现是等差/等比/斐波那契等，直接套公式。

```cpp
// 手算：f(1)=1, f(2)=3, f(3)=6, f(4)=10 ... → f(n)=n(n+1)/2
long long ans = 1LL * n * (n + 1) / 2;
```

## 例 3：构造题

> 要求**构造**一个满足条件的方案（而非判断/计数）。先想一个简单的通用构造，再验证它满足所有约束。

构造题没有唯一答案，**能凑出一个合法解即可**。常用「对称构造」「贪心地一项项放」。

## 心态与策略

- 卡住时**先跳过**，做完其他题再回来。
- 把题目条件**逐条列出**，常有被忽略的关键限制。
- 大胆**猜结论**，再用小样例验证；竞赛允许「猜中即可」。

## 易错提醒

- 别强行套用标准算法——Ad Hoc 的关键是观察。
- 找到规律后，务必用**更多样例验证**（尤其边界 $n=0,1$）。
- 构造题注意所有约束都满足，别只顾主条件。

## 小结

- Ad Hoc 题无固定套路，靠**观察、找规律、构造**解决。
- 方法：手算小样例、找不变量/奇偶性、暴力打表找通项。
- 策略上可先猜后验、卡住先跳过。
