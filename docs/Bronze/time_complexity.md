# 时间复杂度

> **难度**：⭐☆☆☆☆
> **前置知识**：基本循环、数组

时间复杂度用**大 O 记号**描述算法运行时间随数据规模 $n$ 增长的快慢。学会它，才能判断「我的暴力会不会超时」——这是 USACO 的第一课。

## 为什么重要

竞赛评测机大约每秒能做 $10^8 \sim 10^9$ 次基本运算。比赛通常**时限 1~2 秒**。看到数据范围，就能反推允许的复杂度：

| $n$ 的范围 | 可接受复杂度 |
| --- | --- |
| $n \le 10$ | $O(n!)$、$O(2^n)$ |
| $n \le 20$ | $O(2^n)$ |
| $n \le 500$ | $O(n^3)$ |
| $n \le 5000$ | $O(n^2)$ |
| $n \le 10^6$ | $O(n\log n)$、$O(n)$ |
| $n \le 10^{18}$ | $O(\log n)$、$O(1)$ |

> 拿到题先看 $n$，再决定算法。$n \le 1000$ 时 $O(n^2)$ 完全没问题；$n = 10^6$ 时 $O(n^2)=10^{12}$ 必然超时。

## 大 O 记号

大 O 只保留**最高次项**、忽略常数：

- 单层循环 → $O(n)$
- 双层嵌套 → $O(n^2)$
- 每次折半 → $O(\log n)$
- 排序 → $O(n\log n)$
- 固定步数 → $O(1)$

```cpp
// O(n)：单层
for (int i = 0; i < n; i++) sum += a[i];

// O(n^2)：双层
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        if (a[i] + a[j] == target) cnt++;

// O(log n)：每次减半
while (x > 1) x /= 2;
```

## 常见复杂度增长对比

$$O(1) < O(\log n) < O(n) < O(n\log n) < O(n^2) < O(2^n) < O(n!)$$

| $n$ | $n$ | $n\log n$ | $n^2$ |
| --- | --- | --- | --- |
| $10^3$ | $10^3$ | $\sim10^4$ | $10^6$ |
| $10^6$ | $10^6$ | $\sim2\times10^7$ | $10^{12}$ ✗ |

## 空间复杂度

同理描述**内存**用量。USACO 内存限制通常 256 MB，约可存 $6\times10^7$ 个 `int`。开 `int dp[100000][100000]` 会爆内存。

## 估算技巧

把循环次数相乘，再对照「$10^8$/秒」：

```cpp
for (int i = 0; i < n; i++)        // n
    for (int j = 0; j < n; j++)    // ×n
        ...                        // 共 n^2 次
// n=5000 → 2.5×10^7，约 0.1 秒，可行
// n=50000 → 2.5×10^9，超时
```

## 易错提醒

- 大 O 忽略常数，但**实际常数大**（如复杂的内层操作）仍可能超时。
- 嵌套循环次数**相乘**不是相加。
- `std::sort` 是 $O(n\log n)$；别在循环里反复排序导致 $O(n^2\log n)$。

## 小结

- 先看数据范围 $n$，用上表反推允许的复杂度，再选算法。
- 大 O 保留最高次项：单层 $O(n)$、双层 $O(n^2)$、折半 $O(\log n)$。
- 评测机约 $10^8$ 次/秒，是估算超时与否的基准。
