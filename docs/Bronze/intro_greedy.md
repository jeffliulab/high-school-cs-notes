# 贪心入门

> **难度**：⭐⭐☆☆☆
> **前置知识**：[排序入门](intro_sorting.md)

贪心算法**每一步都做当前看起来最优的选择**，期望由局部最优累积出全局最优。它简单高效，但**不是所有问题都适用**——关键是判断贪心是否正确。

## 贪心的思想

不回头、不试错，每步直接选「眼下最好」的：

- 找零钱：每次用面额最大的硬币。
- 活动安排：每次选结束最早的活动。
- 排队接水：让耗时短的人先打水，使总等待最小。

## 经典例：区间调度（最多不重叠活动）

> 给定若干活动的开始、结束时间，选出最多个互不重叠的活动。

**贪心策略：按结束时间排序，每次选能选的、结束最早的。**

```cpp
struct Act { int s, e; };
bool cmp(Act a, Act b) { return a.e < b.e; }  // 按结束时间升序

int main() {
    sort(act, act + n, cmp);
    int cnt = 0, lastEnd = -1;
    for (int i = 0; i < n; i++) {
        if (act[i].s >= lastEnd) {    // 不与上一个重叠
            cnt++;
            lastEnd = act[i].e;       // 选它，更新结束时间
        }
    }
    cout << cnt << "\n";
}
```

直觉：结束越早，给后面留的空间越大。

## 经典例：最小化总等待时间

> $n$ 个人打水，耗时各不同，安排顺序使所有人**总等待时间**最小。

**策略：耗时短的先打。** 排序后，第 $i$ 个人的耗时会被后面 $n-i$ 个人都等到，所以让小的排前面。

```cpp
sort(t, t + n);                 // 升序
long long total = 0, prefix = 0;
for (int i = 0; i < n; i++) {
    total += prefix;            // 前面人打水的累计时间 = 此人的等待
    prefix += t[i];
}
```

## 贪心正确吗？——这是关键

贪心**容易写但容易错**。要么能给出「为什么局部最优导致全局最优」的论证（常用**交换论证**：证明任何最优解都能调整成贪心解而不变差），要么用**小数据暴力对拍**验证。

> 铜级常见做法：先想一个贪心策略，**用几个样例和暴力解对比**，一致再提交。若找到反例，说明贪心错，需换思路（可能要 DP）。

## 常见的「假贪心」陷阱

- 背包问题（容量有限、物品不可分）：按性价比贪心**不一定对**，需 DP。
- 有「后效性」的问题：当前选择影响未来可选项时，贪心常失效。

## 易错提醒

- 别想当然——**贪心需要验证**（交换论证或暴力对拍）。
- 排序的关键字选错（如按开始时间而非结束时间）会得到错误贪心。
- 注意数据溢出，总和用 `long long`。

## 小结

- 贪心 = 每步取局部最优；适用时简单高效。
- 经典模型：区间调度（按结束时间排序）、最小等待（短任务优先）。
- **必须验证贪心的正确性**，否则容易掉进「假贪心」陷阱。
