# 模拟

> **难度**：⭐☆☆☆☆
> **前置知识**：[数据结构入门](intro_data_structures.md)

模拟题就是「按题目描述，一步一步照做」。它是铜级最常见的题型——不需要巧妙算法，只考查**耐心、细致地把过程翻译成代码**。

## 什么是模拟

题目给出一个明确的**过程或规则**，你只需如实地用代码重现它，直接得到答案。难点不在算法，而在：

- 完整、准确地实现每一条规则；
- 处理好边界与特殊情形；
- 不超时（确认朴素模拟的复杂度可接受）。

## 例 1：按指令移动

> 一个机器人在网格上，初始在 $(0,0)$，给定一串指令 `U/D/L/R`，求最终位置。

```cpp
int x = 0, y = 0;
string s;
cin >> s;
for (char c : s) {
    if (c == 'U') y++;
    else if (c == 'D') y--;
    else if (c == 'L') x--;
    else if (c == 'R') x++;
}
cout << x << " " << y << "\n";
```

## 例 2：逐轮模拟过程

> $n$ 头奶牛排队，每分钟队首离开，其余前移……模拟若干分钟后的状态。

直接用数组/队列按分钟推进即可。**只要每步复杂度可控，总步数不大**，朴素模拟就能过。

## 用方向数组简化网格模拟

网格题里上下左右移动，用 `dx/dy` 数组比一堆 if 更整洁：

```cpp
int dx[] = {-1, 1, 0, 0};   // 上下左右
int dy[] = {0, 0, -1, 1};
for (int d = 0; d < 4; d++) {
    int nx = x + dx[d], ny = y + dy[d];
    if (nx >= 0 && nx < R && ny >= 0 && ny < C) {
        // 合法相邻格
    }
}
```

## 模拟题的常见坑

- **读题不全**：漏掉某条规则或边界条件。
- **边界**：网格出界、数组越界、第 0 步/最后一步特判。
- **顺序**：规则的先后顺序影响结果（先移动还是先判定？）。
- **数据范围**：确认模拟步数 × 每步代价不超时；必要时找规律避免硬模拟（如循环节）。

## 复杂度自查

若需要模拟 $T$ 步、每步 $O(n)$，总复杂度 $O(Tn)$。当 $T$ 极大（如 $10^{18}$ 分钟）时不能硬模拟，要找**循环节/周期**或用数学公式跳过。这类「优化的模拟」更偏银级。

## 易错提醒

- 严格按题意，**别自作主张简化规则**。
- 仔细处理出界、空集合、起止时刻等边界。
- 估算总步数，确认不超时再写硬模拟。

## 小结

- 模拟 = 忠实地把题目过程翻译成代码，铜级第一大题型。
- 善用方向数组处理网格移动。
- 重点是**读全规则 + 处理边界 + 确认不超时**。
