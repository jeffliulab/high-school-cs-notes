# Kruskal 重构树

> **难度**：⭐⭐⭐⭐⭐
> **前置知识**：[最小生成树](../Gold/mst.md)、[倍增](binary_jumping.md)

Kruskal 重构树把 Kruskal 建 MST 的过程「树形化」，把「两点路径上的**最大边权**」这类查询，转化为重构树上的 **LCA 点权**查询。

## 构造过程

按 Kruskal 顺序（边权**升序**）逐条加边。每次合并两个连通块时，**新建一个虚拟节点**作为这两个连通块当前代表的**新父亲**，其点权 = 这条边的权值。

```cpp
// 初始 n 个叶子节点；cnt 为当前节点数
int cnt = n;
int val[2*N];           // 节点点权（虚拟节点存边权）
int fa[2*N];            // 重构树父亲
// 并查集 find/unite

sort(edges by weight ascending);
for (auto& e : edges) {
    int x = find(e.u), y = find(e.v);
    if (x != y) {
        int z = ++cnt;          // 新虚拟节点
        val[z] = e.w;           // 点权 = 边权
        treeParent[x] = z;      // 两个连通块的代表挂到 z 下
        treeParent[y] = z;
        dsu[x] = z; dsu[y] = z; // 并查集合并到 z
    }
}
// 最终根是最后建的虚拟节点
```

结果是一棵**二叉树**：$n$ 个原始点是叶子，$n-1$ 个虚拟节点是内部点，**点权从根到叶递减**（大根堆性质）。

## 核心性质

两点 $u, v$ 在原图中**路径上的最大边权的最小值**（即「瓶颈路」）= 重构树上 **$\text{LCA}(u, v)$ 的点权**。

> 直观：按升序加边，$u$、$v$ 第一次连通时所加的那条边就是它们瓶颈路的最大边，而那条边对应的虚拟节点正是它们在重构树上的 LCA。

## 应用

- **瓶颈路**：两点间使「最大边权最小」的路径，其值 = LCA 点权。
- **「只走边权 ≤ w 的边能到达哪些点」**：在重构树上，从某点向上跳到「点权 ≤ w 的最高祖先」，其**子树叶子**就是可达点集合（配合 [倍增](binary_jumping.md) 跳 + [欧拉序](../Gold/euler_tour.md) 取子树）。
- 配合主席树可做「第 K 小可达点」等。

## 与最大生成树版本

若按**降序**加边建重构树，则 LCA 点权 = 两点路径**最小边权的最大值**（最大瓶颈路），用于「最大化路径最小边」类问题。

## 易错提醒

- 节点数组开 `2*N`（叶子 + 虚拟节点共约 $2n-1$）。
- 升序建树对应「最小化最大边」；降序对应「最大化最小边」。
- 虚拟节点点权 = 边权；叶子点权可视题目设（常为 0 或点本身属性）。
- 查询前对重构树预处理 LCA（倍增）、欧拉序（取子树）。

## 小结

- Kruskal 重构树把建 MST 过程变成二叉树：虚拟节点存边权，点权从根到叶递减。
- **两点瓶颈路 = LCA 点权**；「边权 ≤ w 可达集合」= 跳到合适祖先的子树。
- 升序/降序分别对应最小化最大边 / 最大化最小边。
