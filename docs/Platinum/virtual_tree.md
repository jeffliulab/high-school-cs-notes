# 虚树

> **难度**：⭐⭐⭐⭐⭐
> **前置知识**：[倍增](binary_jumping.md)（LCA）、[树形 DP 入门](../Gold/dp_on_trees.md)

虚树（Virtual Tree）针对「多次询问，每次只关心树上**少数关键点**」的问题：只保留关键点及其两两 LCA，构出一棵小树，在小树上做 DP。当所有询问的关键点总数为 $\sum k$ 时，复杂度 $O(\sum k \log n)$ 而非每次 $O(n)$。

## 适用场景

题目给多组询问，每组指定 $k$ 个关键点，要在原树上做某种 DP/统计。若 $\sum k$ 远小于 $q \cdot n$，但每次都遍历整棵树太慢，就用虚树：把无关的点压缩掉，只在「关键点 + 它们的 LCA」构成的虚树上计算。

## 构建虚树

1. 把关键点按 **DFS 序（dfn）** 排序。
2. 相邻关键点求 **LCA**，这些 LCA 也是虚树节点。
3. 用**单调栈**沿 DFS 序加入节点，连出虚树的边（边权为原树上两点距离）。

```cpp
// key[] 为关键点；按 dfn 排序后：
sort(key, key + m, [](int a, int b){ return dfn[a] < dfn[b]; });
// 把相邻关键点的 LCA 也加入节点集合，去重，再按 dfn 排序
// 用栈构建：维护从根往下的一条链，新点与栈顶求 LCA 决定连边
stack<int> stk;
stk.push(root);
for (each node u in sorted order) {
    int l = lca(u, stk.top());
    while (栈中比 l 深的点) {
        // 弹栈并连边到 l 或更上层
    }
    addEdge(l, u);
    stk.push(u);
}
```

（具体实现有多种写法，核心是「按 dfn 顺序 + LCA + 栈」连边。）

## 在虚树上 DP

虚树的节点数为 $O(k)$，边带权（原树距离）。原本在整棵树上的树形 DP，现在只在这 $O(k)$ 个节点上做，关键点之间的「链」用边权代表，非关键路径被压缩。

## 复杂度

每次询问构建虚树 $O(k\log n)$（排序 + LCA），DP $O(k)$。总 $O(\sum k \log n)$。

## 应用

- 「每次给若干关键点，求把它们与根/彼此连通的最小代价」。
- 关键点之间的路径统计、关键点子树 DP。
- 多组询问的树形 DP 加速。

## 易错提醒

- 关键点和插入的 LCA 都要去重并按 `dfn` 排序。
- 虚树的边权是原树上两端点的**距离**（深度差或路径和），别当成 1。
- 每次询问后要**只清空**用到的虚树节点（不能 memset 整棵原树，否则失去复杂度优势）。
- 根节点是否要强制加入视题目而定。

## 小结

- 虚树只保留**关键点 + 两两 LCA**，把大树压成 $O(k)$ 的小树。
- 构建：按 dfn 排序 + LCA + 单调栈连边（边权为原树距离）。
- 在虚树上做树形 DP，总复杂度 $O(\sum k\log n)$；注意按需清空。
