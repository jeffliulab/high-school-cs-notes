# bitset

> **难度**：⭐⭐⭐⭐☆
> **前置知识**：[位运算入门](../Silver/bitwise.md)

`std::bitset` 把布尔数组压成位，使位运算**并行处理 64 位**，带来 $1/64$ 的常数加速。它能让某些 $O(n^2)$ 算法在 $n \le 5\times10^4$ 时跑过。

## 基本用法

```cpp
#include <bitset>
bitset<100005> b;         // 大小须是编译期常量
b.set(3);                 // 第 3 位置 1
b.reset(3);               // 第 3 位置 0
b[5] = 1;                 // 直接下标赋值
bool x = b[5];            // 读取
b.flip();                 // 全部取反
int c = b.count();        // 1 的个数
bool any = b.any();       // 是否有 1
```

## 位运算（核心价值）

两个 `bitset` 的 `& | ^` 一次处理所有位，实际是按 64 位一组并行：

```cpp
bitset<N> a, b;
a |= b;       // 并
a &= b;       // 交
a ^= b;       // 对称差
a <<= k;      // 整体左移 k 位
```

$N$ 位的运算耗时 $O(N/64)$ 而非 $O(N)$。

## 应用 1：可达性 / 子集和加速

> 判断能否用一些数凑出每个值（背包可行性）。

```cpp
bitset<MAXSUM> dp;
dp[0] = 1;
for (int x : nums) dp |= (dp << x);   // 每个数把已有可达和平移 x
// dp[s] 为 1 表示 s 可凑出
```

把 $O(nW)$ 的背包用 bitset 加速到 $O(nW/64)$。

## 应用 2：传递闭包 / 图可达

> 求每个点能到达哪些点（Floyd 式传递闭包）。

```cpp
bitset<N> reach[N];          // reach[i] 第 j 位 = i 能否到 j
for (int k = 0; k < n; k++)
    for (int i = 0; i < n; i++)
        if (reach[i][k]) reach[i] |= reach[k];   // 一次合并整行
```

把 $O(n^3)$ 降到 $O(n^3/64)$，$n$ 可到约 2000+。

## 应用 3：字符串匹配 / 最长公共子序列

bitset 优化的 LCS（Hunt–Szymanski / 位并行）可处理较大规模。

## 复杂度直觉

bitset 不改变渐进复杂度，但**常数除以 64**。原本 $n^2 = (5\times10^4)^2 = 2.5\times10^9$ 超时，除以 64 后约 $4\times10^7$，可过。这是「卡常」利器。

## 易错提醒

- `bitset<N>` 的 $N$ 必须是**编译期常量**，不能用变量。
- 大小固定，不能动态调整；开太大费内存/时间。
- 下标范围 $[0, N)$，越界是未定义行为。
- 移位 `<<= k` 后高位溢出会丢弃。

## 小结

- `bitset` 把布尔数组压位，位运算**并行 64 位**，常数 $/64$。
- 经典加速：背包可行性（`dp |= dp<<x`）、传递闭包、LCS。
- 不改变渐进复杂度，是把 $O(n^2)$/$O(n^3)$ 卡进时限的利器。
