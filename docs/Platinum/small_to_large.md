# 启发式合并

> **难度**：⭐⭐⭐⭐☆
> **前置知识**：[树算法入门](../Silver/intro_trees.md)、[集合与映射](../Bronze/sets_maps.md)

启发式合并（小到大合并 / DSU on tree）在合并集合时**总把小的并入大的**，保证总复杂度 $O(n\log n)$。常用于「对每棵子树统计信息」。

## 核心原理

合并两个集合时，把元素少的那个**逐个插入**元素多的那个。关键结论：每个元素**最多被「搬动」$O(\log n)$ 次**——因为每次它所在的集合至少**翻倍**（搬到的集合不小于原集合）。故总搬动次数 $O(n\log n)$。

## 集合启发式合并

```cpp
set<int> s[N];           // 每个节点一个集合

void merge(int a, int b) {        // 把 b 并入 a
    if (s[a].size() < s[b].size()) swap(s[a], s[b]);  // 保证 a 更大
    for (int x : s[b]) s[a].insert(x);                // 小的并入大的
    s[b].clear();
}
```

`swap` 两个 `set` 是 $O(1)$（只换内部指针），这是保证复杂度的关键。

## 树上启发式合并（DSU on tree）

对每个节点，统计其**子树**的某种信息（如出现次数最多的颜色、不同颜色数）。朴素地对每棵子树重算是 $O(n^2)$；启发式合并做到 $O(n\log n)$。

```cpp
// 自底向上：把所有子节点的信息合并到当前节点
map<int,int> cnt[N];     // 或用全局结构 + 重儿子保留

void dfs(int u, int p) {
    cnt[u][color[u]] = 1;
    for (int v : adj[u]) {
        if (v != p) {
            dfs(v, u);
            // 小到大合并 cnt[v] 到 cnt[u]
            if (cnt[u].size() < cnt[v].size()) swap(cnt[u], cnt[v]);
            for (auto& [c, k] : cnt[v]) cnt[u][c] += k;
        }
    }
    // 此时 cnt[u] 是整棵子树 u 的统计，回答关于 u 的查询
}
```

## 更优：重儿子复用（经典 DSU on tree）

进阶写法用全局计数数组，先递归轻儿子（算完清空），再递归**重儿子并保留其贡献**，最后把轻儿子重新加入。同样 $O(n\log n)$，常数更小，适合需要全局 cnt 数组的场景。

## 应用

- 子树内出现次数最多的颜色 / 不同颜色数。
- 子树内某种配对计数。
- 维护每个节点子树的有序集合（求中位数、第 K 大）。

## 易错提醒

- **必须小并入大**，且用 `swap` 交换容器（$O(1)$），否则退化 $O(n^2)$。
- `swap` 后注意「合并方向」可能变了，统一以「大的」为结果容器。
- 用 `map`/`set` 合并时复杂度多一个 $\log$（总 $O(n\log^2 n)$）；用数组 + 重儿子复用可去掉。
- 合并后清空被并入的容器，避免重复。

## 小结

- 启发式合并：合并时**小集合并入大集合**，每元素搬动 $O(\log n)$ 次。
- 用 `swap` 容器保证 $O(1)$ 交换，总 $O(n\log n)$（map 则多一个 log）。
- 树上版（DSU on tree）高效统计每棵子树信息。
