# 扫描线区间查询

> **难度**：⭐⭐⭐⭐☆
> **前置知识**：[线段树进阶应用](segtree_applications.md)、排序

扫描线把**二维问题降为一维**：用一条线沿某个维度扫过，在每个事件点用数据结构（线段树/BIT）维护另一维的状态。常用于离线区间查询。

## 核心思想

把所有操作和查询按某个坐标（如 $x$ 或时间）排序，想象一条扫描线从左到右移动。线扫到一个「事件」时更新数据结构；扫到一个「查询」时直接读取当前状态。这样把对二维的处理转化为「一维上的若干次更新与查询」。

## 经典：离线「矩形加，单点查」/「区间数颜色」

> 给若干带时间的区间操作和查询，离线后按坐标排序处理。

**例：区间不同元素个数（HH的项链）。** 把查询按右端点排序，扫描线右移；对每个新元素，只在它**最后一次出现的位置**计数（用 BIT 维护），查询时求区间和。

```cpp
// 查询按 r 排序，i 从 0 扫到 n-1
sort(queries by r);
int last[VAL];           // 每个值上次出现的位置，初始 -1
for (int i = 0; i < n; i++) {
    int v = a[i];
    if (last[v] != -1) bit.update(last[v], -1);  // 撤销旧位置
    bit.update(i, 1);                            // 当前位置 +1
    last[v] = i;
    // 处理所有 r == i 的查询：答案 = bit.rangeSum(l, i)
    for (auto& q : queriesEndingAt[i])
        ans[q.id] = bit.rangeSum(q.l, i);
}
```

## 经典：矩形面积并（扫描线 + 线段树）

> 求若干轴对齐矩形的并集面积。

把每个矩形拆成「左边界（加）」和「右边界（减）」两个事件，按 $x$ 排序。扫描线沿 $x$ 移动，用线段树维护当前被覆盖的 $y$ 区间**总长度**；每段 $x$ 贡献面积 = 当前覆盖长度 × $x$ 跨度。

```cpp
// 事件：(x, y1, y2, +1/-1)，按 x 排序
sort(events);
long long area = 0;
for (int i = 0; i < events.size(); i++) {
    if (i > 0) area += segtree.coveredLength() * (events[i].x - events[i-1].x);
    segtree.update(events[i].y1, events[i].y2, events[i].type);  // 区间 +1/-1
}
```

这里线段树维护「每个 $y$ 区间被覆盖几次、总覆盖长度」，是扫描线的标志题。

## 离线 vs 在线

扫描线通常是**离线**的（需要先收集所有查询再排序）。若强制在线，往往需要可持久化数据结构。

## 易错提醒

- 事件和查询要按扫描维度正确排序；相同坐标的事件处理顺序需小心（先加后查 or 反之）。
- 区间数颜色类问题，要用「只保留最后出现位置」的技巧。
- 面积并的线段树维护「覆盖次数 + 覆盖长度」，离散化 $y$ 坐标。
- 坐标大时先离散化。

## 小结

- 扫描线：按一维排序，线扫过时用数据结构维护另一维，把二维降一维。
- 经典：区间数颜色（BIT + 最后出现位置）、矩形面积并（线段树维护覆盖长度）。
- 多为离线算法，注意事件排序与离散化。
