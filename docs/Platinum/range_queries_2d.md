# 二维区间查询

> **难度**：⭐⭐⭐⭐⭐
> **前置知识**：[单点修改区间和](../Gold/point_update_range_sum.md)、[扫描线区间查询](sweep_line_range.md)

把区间查询推广到二维平面：查询某个矩形内的和/点数，支持修改。根据是否有修改、是否在线，有二维 BIT、二维线段树、离线扫描线等方案。

## 静态：二维前缀和

无修改时最简单——[二维前缀和](../Silver/prefix_sums_advanced.md) 容斥 $O(1)$ 查矩形和。本页关注**带修改**的情形。

## 二维树状数组

支持「单点修改 + 矩形和查询」，是二维 BIT 的直接推广：

```cpp
long long tree[N][N];
void update(int x, int y, long long v) {
    for (int i = x; i <= n; i += i & -i)
        for (int j = y; j <= m; j += j & -j)
            tree[i][j] += v;
}
long long query(int x, int y) {          // 前缀矩形 [1..x][1..y]
    long long s = 0;
    for (int i = x; i > 0; i -= i & -i)
        for (int j = y; j > 0; j -= j & -j)
            s += tree[i][j];
    return s;
}
// 矩形 [x1..x2][y1..y2] 容斥
long long rect(int x1,int y1,int x2,int y2){
    return query(x2,y2) - query(x1-1,y2) - query(x2,y1-1) + query(x1-1,y1-1);
}
```

复杂度每次 $O(\log n \log m)$，空间 $O(nm)$——要求坐标范围不大（如 $\le 2000$）。

## 坐标大：离线扫描线 + BIT

坐标到 $10^9$ 时无法开 $nm$ 数组。把矩形查询用容斥拆成 4 个「前缀矩形」查询，按 $x$ **扫描线**，对 $y$ 用一维 BIT（先离散化 $y$）：

> 这是「二维数点」的标准离线做法：查询 $(x \le X, y \le Y)$ 的点数 → 按 $x$ 排序点和查询，扫到 $x \le X$ 时把点插入 $y$ 的 BIT，查询时求 $y \le Y$ 的前缀。

## 在线大坐标：树套树 / 主席树

- **BIT 套动态开点线段树**（或线段树套平衡树）：支持在线，空间 $O(q\log^2)$。
- **主席树**：对「静态二维数点 / 区间第 K 小」很常用，按一维建版本，另一维在权值线段树上查。

## 方案选择

| 场景 | 方案 |
| --- | --- |
| 无修改、坐标小 | 二维前缀和 $O(1)$ |
| 单点改、坐标小 | 二维 BIT |
| 坐标大、离线 | 扫描线 + 一维 BIT（离散化） |
| 坐标大、在线 | 树套树 / 主席树 |

## 易错提醒

- 二维 BIT 空间 $O(nm)$，坐标大会爆——此时必须扫描线或离散化。
- 矩形查询用二维容斥（4 项），符号别错。
- 离线扫描线要正确排序点与查询、离散化另一维。
- 全程 `long long`。

## 小结

- 二维区间查询按「有无修改 / 坐标大小 / 在线离线」选方案。
- 坐标小带修改用**二维 BIT**（$O(\log n\log m)$）。
- 坐标大优先**离线扫描线 + 一维 BIT**；在线则树套树/主席树。
