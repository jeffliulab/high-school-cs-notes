# 分治求静态区间查询

> **难度**：⭐⭐⭐⭐⭐
> **前置知识**：[分治](../Gold/meet_in_middle.md)、离线处理

对于**静态**（无修改）的离线区间查询，CDQ 分治 / 「按区间跨中点分治」能在 $O((n+q)\log n)$ 解决许多用普通数据结构难处理的问题。

## 核心思想：按中点分治

把序列从中点 $mid$ 分成左右两半，递归处理「完全落在左半」和「完全落在右半」的查询；对**跨越中点**的查询，单独用一遍线性扫描统一处理。

```
solve(l, r, 查询集):
    if 查询集为空 or l == r: 直接处理
    mid = (l + r) / 2
    把查询分成三类：
      - 完全在 [l, mid]  → 递归 solve(l, mid, ...)
      - 完全在 [mid+1, r]→ 递归 solve(mid+1, r, ...)
      - 跨越 mid          → 在本层统一计算
    跨越中点的查询：从 mid 向左、向右各扫一遍预处理，O(区间长) 回答
```

跨中点的查询是关键：因为它们都包含 $mid$，可以预处理「从 $mid$ 向左到各点」「向右到各点」的某种前缀信息，再 $O(1)$ 合并。

## 例：静态区间「最大子段和」查询

> 多次询问 $[l, r]$ 内的最大连续子段和，无修改。

跨中点的最大子段和 = 「左半以 mid 结尾的最大后缀和」+「右半从 mid+1 开始的最大前缀和」类组合。在分治每层，对包含 mid 的查询用预处理的前后缀信息合并。

## CDQ 分治（三维偏序）

更广义的「分治套分治」用于**三维偏序**：统计满足 $a_i \le a_j, b_i \le b_j, c_i \le c_j$ 的对数。

1. 第一维**排序**。
2. 第二维**CDQ 分治**：处理「左半对右半」的贡献。
3. 第三维用 **BIT** 统计。

总复杂度 $O(n\log^2 n)$。它是「带修改的二维数点」「偏序计数」的通用框架，可替代树套树且常数更小。

## 与其他方法对比

- **莫队算法**：另一种处理静态区间查询的离线方法（按块排序查询，$O((n+q)\sqrt n)$），实现更简单，适合「区间数颜色」等。
- **主席树**：在线静态区间第 K 小等。
- **CDQ/分治**：偏序、需要按维度递推的问题，$O(n\log^2 n)$。

## 易错提醒

- 分治必须**离线**（先收集所有查询）。
- 跨中点的处理是正确性关键，预处理前后缀要覆盖所有跨越情况。
- CDQ 三维偏序中，第二维分治时左右两部分要按第三维有序（归并），BIT 用完记得清空。
- 注意去重（相等元素的偏序计数）。

## 小结

- 分治静态区间查询：按中点分，递归两侧 + 单独处理**跨中点**查询。
- **CDQ 分治**解三维偏序：排序 + 分治 + BIT，$O(n\log^2 n)$。
- 均为离线方法；与莫队、主席树互为补充。
