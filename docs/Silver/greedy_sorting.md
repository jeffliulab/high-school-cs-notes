# 排序贪心

> **难度**：⭐⭐⭐☆☆
> **前置知识**：[贪心入门](../Bronze/intro_greedy.md)、[自定义比较器与坐标压缩](custom_comparators.md)

银级最常见的贪心形态：**先按某个关键字排序，再线性扫描做贪心决策**。难点是找到「按什么排序」这个关键洞察，并能论证其正确性。

## 套路

1. 观察问题，猜测一个「排序关键字」。
2. 排序后从前到后贪心地处理每个元素。
3. 用**交换论证**或小数据对拍验证策略正确。

## 例 1：区间调度（按结束时间）

最多选互不重叠的区间——按**结束时间**升序排，每次选不冲突的：

```cpp
sort(itv.begin(), itv.end(), [](auto& x, auto& y){
    return x.second < y.second;       // 按结束时间
});
int cnt = 0, lastEnd = -2e9;
for (auto& [s, e] : itv) {
    if (s >= lastEnd) { cnt++; lastEnd = e; }
}
```

## 例 2：按差值/比值排序

> 每个任务有截止时间和耗时，安排顺序最小化某代价——常按「截止时间」或「耗时」或二者的比值排序。

**经典：最小化最大延迟，按截止时间排序（EDF）。**

```cpp
sort(tasks.begin(), tasks.end(), [](auto&a, auto&b){
    return a.deadline < b.deadline;
});
long long t = 0, maxLate = 0;
for (auto& task : tasks) {
    t += task.len;
    maxLate = max(maxLate, t - task.deadline);
}
```

## 例 3：配对最小化/最大化

> 两组数配对，使配对差之和最小——**两组各自排序后顺次配对**。

```cpp
sort(a, a + n);
sort(b, b + n);
long long cost = 0;
for (int i = 0; i < n; i++) cost += abs(a[i] - b[i]);
```

直觉（交换论证）：若存在交叉配对，交换回顺次配对不会更差。

## 交换论证（证明贪心正确）

核心思路：假设存在一个最优解与贪心解不同，找到第一处不同，**交换**使它更接近贪心解且不变差。反复交换可把任意最优解变成贪心解 → 贪心也是最优。竞赛中能想通这个论证，就敢放心提交。

## 何时贪心会错

- 选择有**后效性**（当前选择限制未来更优选择）。
- 需要全局权衡而非局部最优（常需 DP）。

不确定时，**写暴力对拍**是最稳妥的验证。

## 易错提醒

- 排序关键字选错 = 整个贪心错；多想几种关键字并验证。
- 累加量用 `long long`。
- 贪心结论先用小样例 / 暴力验证再提交。
- 注意排序的稳定性是否影响结果（通常不影响最优值）。

## 小结

- 排序贪心 = **选对关键字排序 + 线性扫描贪心**。
- 经典：区间调度（结束时间）、最小延迟（截止时间 EDF）、顺次配对。
- 用**交换论证**或暴力对拍确认正确性，避免假贪心。
