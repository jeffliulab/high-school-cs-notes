# 双指针

> **难度**：⭐⭐⭐☆☆
> **前置知识**：[排序入门](../Bronze/intro_sorting.md)

双指针用两个一起移动的下标，把许多 $O(n^2)$ 的枚举降到 $O(n)$。常见两种形态：**同向滑动窗口**和**对撞指针**。

## 形态一：对撞指针（首尾向中间）

两个指针从两端向中间逼近，常用于**有序数组**找配对：

> 有序数组中找两数之和等于 $T$。

```cpp
int lo = 0, hi = n - 1;
while (lo < hi) {
    int s = a[lo] + a[hi];
    if (s == T) { /* 找到 */ break; }
    else if (s < T) lo++;       // 和太小，左指针右移增大
    else hi--;                  // 和太大，右指针左移减小
}
```

每步移动一个指针，$O(n)$ 完成，优于 $O(n^2)$ 双层枚举。

## 形态二：滑动窗口（同向）

两个指针同向移动，维护一个「窗口」$[l, r]$ 及其性质：

> 求和 $\le K$ 的**最长**连续子数组长度（元素非负）。

```cpp
int l = 0, sum = 0, best = 0;
for (int r = 0; r < n; r++) {
    sum += a[r];                    // 右指针扩张
    while (sum > K) {               // 超了就收缩左边界
        sum -= a[l];
        l++;
    }
    best = max(best, r - l + 1);    // 当前合法窗口长度
}
```

每个元素**最多被左、右指针各扫一次**，所以是 $O(n)$。

## 关键：为什么是 O(n)

虽然有嵌套的 `while`，但 `l` 全程只增不减、最多走 $n$ 步，`r` 也走 $n$ 步——总移动次数 $O(n)$，不是 $O(n^2)$。这叫**摊还分析**。

## 适用条件

滑动窗口要求窗口性质有**单调性**：右指针右移使窗口「更难满足」时，左指针只需右移收缩，不必回退。例如「和 ≤ K」「不同元素 ≤ K 个」。元素有负数时和不单调，滑动窗口失效。

## 例：合并两个有序数组

双指针也用于归并：各指一个数组，每次取较小者前进。

```cpp
int i = 0, j = 0;
while (i < n && j < m) {
    if (a[i] <= b[j]) res.push_back(a[i++]);
    else res.push_back(b[j++]);
}
// 收尾把剩余的接上
```

## 易错提醒

- 对撞指针要求数组**有序**，否则先排序。
- 滑动窗口的收缩条件、窗口长度 `r-l+1` 别算错。
- 确认问题有单调性，否则滑动窗口会漏解。
- 指针移动后及时更新维护的量（和、计数等）。

## 小结

- **对撞指针**：有序数组首尾逼近找配对，$O(n)$。
- **滑动窗口**：同向移动维护区间性质，靠摊还做到 $O(n)$。
- 适用前提是窗口性质有单调性；有负数等破坏单调时不适用。
