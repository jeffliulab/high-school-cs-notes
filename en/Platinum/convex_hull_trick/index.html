    斜率优化

    难度：⭐⭐⭐⭐⭐ 前置知识：凸包、DP 入门

    斜率优化（Convex Hull Trick, CHT）把形如 \(dp[i] = \min_j(a_j \cdot x_i + b_j)\) 的 DP 转移从 \(O(n^2)\) 优化到 \(O(n\log n)\) 或 \(O(n)\)——本质是维护一组直线，快速查询某 \(x\) 处的最小（大）值。

    适用的 DP 形式

    转移可写成「一组直线在某点取最值」：

    \[dp[i] = \min_{j < i}\big(k_j \cdot x_i + b_j\big)\]

    其中 \(k_j, b_j\) 只依赖 \(j\)，\(x_i\) 只依赖 \(i\)。每个 \(j\) 对应一条直线 \(y = k_j x + b_j\)，求 \(dp[i]\) 就是问「这些直线在 \(x = x_i\) 处的最小值」。

    几何直观

    所有直线的下包络（求最小值时）是一个下凸壳。查询某 \(x\) 的最小值 = 在凸壳上找对应线段。加入直线维护凸壳、查询在凸壳上定位——这正是「凸包」思想用在直线上。

    单调情形：O(n) 双端队列

    当插入直线的斜率单调且查询的 \(x\) 单调时，用单调队列维护凸壳，均摊 \(O(1)\)：

    // 维护下凸壳，存直线 (k, b)
deque<pair<long long,long long>> hull;

bool bad(line a, line b, line c) {
    // c 使 b 变得多余（凸性判断，用叉积式比较避免除法）
    return (b.b - a.b) * (a.k - c.k) >= (c.b - a.b) * (a.k - b.k);
}
long long f(pair<long long,long long> ln, long long x){ return ln.first*x + ln.second; }

// 加入新直线时，从队尾弹出被覆盖的；
// 查询单调 x 时，从队首弹出不再最优的。

    一般情形：李超线段树

    当斜率或查询 \(x\) 不单调时，用 李超线段树（Li Chao Tree）：在值域上维护「每段哪条直线最优」，支持任意插入直线、任意点查询最值，\(O(\log V)\)。

    // 李超树每个节点存「在该区间中点处最优」的直线，插入时递归下放
// 查询点 x：沿路径取所有节点直线在 x 的最值

    经典题型

    • 任务划分/打印：\(dp[i] = \min_j(dp[j] + (\text{sum}_i - \text{sum}_j)^2 + C)\)，展开后是直线形式。
    • 玩具装箱、序列分段等含平方项的 DP。
    • 把含 \(x_i \cdot x_j\) 交叉项的转移整理成 \(k_j x_i + b_j\) 即可套用。

    推导套路

    把转移式展开，分离出「只含 \(i\)」「只含 \(j\)」「\(i,j\) 乘积」三类项，令乘积项的 \(i\) 因子为 \(x_i\)、\(j\) 因子为斜率 \(k_j\)，常数项为 \(b_j\)，化成直线最值形式。

    易错提醒

    • 凸性判断用乘法代替除法，防精度问题与溢出（用 long long，必要时 __int128）。
    • 求最小值维护下凸壳、最大值维护上凸壳，方向别搞反。
    • 斜率/查询是否单调决定用双端队列还是李超树。
    • 推导直线形式时项的归类容易出错，仔细分离 \(i\)、\(j\)。

    小结

    • 斜率优化把 \(dp[i]=\min_j(k_j x_i + b_j)\) 从 \(O(n^2)\) 降到 \(O(n)\)/\(O(n\log n)\)。
    • 几何上是维护直线的凸壳并查询某点最值。
    • 斜率与 \(x\) 单调用单调队列；否则用李超线段树。
